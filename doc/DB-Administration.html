DB – Administration

In diesem Dialog werden Methoden verwaltet die direkt mit der Datenbank zusammen arbeiten.
Datensicherung, ADIF-Import -Export, Verwaltung der Tabellen-Layoutnamen.

Datensicherung:
----------------------
Auswahlhaken : "Datenbank komplett sichern“.
DUMP schreibt alle Log- und System Tabellen in eine Datei.
Hierfür muss ein Pfad für einen Ordner der die Sicherungsdatei aufnehmen soll angegeben werden.

RESTORE stellt die mit DUMP exportierten Tabellen wieder in die Datenbank zurück und überschreibt hierbei die Tabellen in der Datenbank.

Auswahl : Logtabelle
DUMP schreibt nur die im Logbook Ordner markierte Logtabelle in eine Textdatei. Der Pfad für diesen Ordner der die Sicherungsdatei aufnehmen soll muss angegeben sein.

RESTORE schreibt die LOG-Daten wieder zurück. Der Name der Logtabelle die überschrieben werden soll muss im Logordner vorhanden und markiert sein.

ADIF -Import / -Export
---------------------------------
IMPORT : Die Logbook-Tabelle die den Import aufnehmen soll im Logbook-Ordner markieren. RIG-Auswahl und den Portabel Standort einstellen. IMPORT durchführen.

EXPORT : Die Logbook-Tabelle die exportiert werden soll markieren. Landespräfix als ADIF-Nr. exportieren vorgeben. EXPORT durchführen.

Layout-Namen
--------------------
Mit dieser Tabelle werden die Spaltennamen für diverse Datentabellen verwaltet. Ebenso die ADIF-Namen die zur Übersetzung  der Datenfelder beim Export/Import benötigt werden. Die Namen lassen sich alle verändern, aber VORSICHT hier kann viel kaputt gehen.
