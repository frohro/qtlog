Contest Logging

Der Dialog Contest Logging stellt einen simplen Eingabe Dialog für einige Contest-Veranstaltungen zur Verfügung.

Eingerichtet wurden z.Z lediglich nur die wichtigsten KW – Conteste für die Betriebsarten SSB und CW. Der Dialog ist noch in der Entwicklung. Die Contest – Modelle werden mit jeder Contest-Veranstaltung weiter vervollständigt.
Beim Programm Aufruf meldet sich ein blauer Bildschirm mit dem Auswahl-Menü Contest.

Button New:
Im allgemeinen wird als erstes ein neuer Contest angelegt → NEW HF-CW. Hier stehen ca. 10 Contest-Modelle zur Auswahl. Hinter jedem Contestmodell sind die wichtigsten Contestvorgaben hinterlegt. Wird auf ein Contesmodell geklickt werden für diesen Contest die Qso-Punkte und Multiplikator angezeigt. Die Bandauswahl wird gesetzt. Es ist aber immer zweckmäßig diese mit der Contestausschreibung abzustimmen. Die Bandauswal ist eine Information und wird nicht im Contest-Dialog verwaltet.

Mit dem Button Create wird der Contest angelegt. Die Contest_id besteht aus dem Contest-Kürzel und der Jahreszahl des laufenden Jahres. Falls erforderlich kann kann die Contest_id mit einem kurzem Anhang erweitert werden. Es wird eine Exchange Info ausgegeben die bestätigt wird. Das Info Widget Sent exchange gibt einen Hinweis für die erforderliche Modifikation der sent variable. zB. beim DARC10M Contest wird sent mit dem DOK ergänzt.

Dedienungs und Informations Elemente:
Contest Ergebnis Übersicht: Hier wird der aktuelle Conteststand je Band dargestellt.
Anzahl Qsos, Points Punkte je QSO , Punkte Multi1, Punkte Multi2.
QSOsh und Score werden z.Z nicht berechnet bzw. hoch gerechnet.
Band: Im Fenster Band wird das aktive Band in kHz angezeigt. Ist RigCtl freigegeben folgt die Anzeige der Transceiverfrequenz.
Button Mode: SSB oder CW. Ist RigCtl freigegeben wird die Betriebsart vom Transceiver geschaltet.
Button Edit: Anzeige aller QSOs für diesen Contest. Weitere Funktionen gibt es für diesen Dialog z.Z. nicht.
Button Info: Anzeige der Exchange Bedingungen.

Keyboard:
ESC - löscht die Eingabe und setzt den Cursor in Callsign
Space ( leer ) - toggelt den Eingabecursor zwischen Callsign und Exchange
Taste Pfeil rauf – Pfeil runter – schaltet das Band vor oder zurück. Ist RigCtl freigegeben wird auch der Transceiver geschaltet.

Logging - im neuen Contest:
Das Programm reagiert auf jede Tastatureingabe im Callsign Fenster. Werden bei der Rufzeichen Eingabe bereits geloggte und zur Eingabe passende QSOs gefunden werden diese angezeigt. Wird ein identisches Rufzeichen gefunden wird dieses QSO rot markiert, als bereits gearbeitet. Selektiert wird je nach Contestmodell, auf Bandebene oder über alle Bänder. Im Runtimefenster stehen immer die zuletzt geloggten max. 20 QSOs.

Korrekturen am Qso:
Mit einem Doppelklick auf eine QSO Zeile wird dieses in den Editmodus gehoben. Der Button Remove QSO wird freigegeben und eine Info in rot zeigt Edit mode. Das QSO kann jetzt editiert werden. Alle Änderung sind zulässig. Das Rufzeichen kann auch geändert bzw. korrigiert werden. Mit 'cr' im Exchange Fenster werden die Korrekturen übernommen. Mit dem Button Remove QSO wird das QSO gelöscht. Qso-Punkt und Multiplikator-Punkte werden vor bzw. zurück gerechnet.

Continue - fortführen eines vorhandenen Contest
Beim Aufruf werden alle Contest-Modelle die in der Contesttabelle stehen angezeigt. Mit einem Klick in die Zeile wird der gewünscht Contest selektiert, mit dem Button Continue wird der Contest geladen, mit ESC wird ohne Aktion zurück geschaltet.

Der Contest kann jetzt weiter bearbeitet werden, alle Funktionen stehen zur Verfügung. Zeit oder Datumsgrenzen ( Contest out of time ) sind nicht vorhanden.

Sent Log ( Cabrillo, STF ) - Logdatei an den Contestveranstalter senden
Beim Aufruf werden die vorhandenen geloggten Conteste angezeigt. Mit einem Klick wird der Contest selektiert. Den Button Sent Log … aufrufen. Der Dialog schaltet zu den Details für den Datenexport. Zur Zeit gibt es für jeden Contest nur ein Exportformat. Conteste die vom DARC durchgeführt werden, werden mit dem Format nach Stuezenbach ( STF ) exportiert, die übrigen mit dem Carillo Format. Der Export im Plain text Format geht, aber ist nicht immer durchführbar, es kommt eine Meldung. Der Dateipfad für die zu exportierend Datei festlegen und und mit dem Button OK bestätigen.

Im folgenden Dialog sind die contest typischen Angaben zum Contest und Operator festzulegen. Mit OK wird die Exportdatei erzeugt. Die exportierten QSO Daten können mit einem Editor angesehen werden.

Convert to QtLog – Contest QSOs in die Datenbank übernehmen
Auch bei dieser Funktion muss als erstes der Contest selektiert werden Im folgenden ContestLog Tabelle wird mit dem Button Convert to QtLog Database die Übertragung durchgeführt.

Hinweis ! Bei der ersten Übertragung der Contest-QSOs in die QSO-Datenbank sollte man vorsorglich die Datenbank auf die Logtabelle fun stellen und eine Testübertragung durchführen.

Delete Contest Log – Contestlog in der Contestabelle löschen
Als erstes wird der zu löschende Contest selektiert. Es erfolgt eine Rückfrage, bei Bestätigung wird der Contest gelöscht.

CW Keyer – einbinden
Der cwdaemon muss installiert sein. Bei den meisten Distrubutionen ist der Daemon auf der DVD oder er kann über das Inernet installiert werden. Siehe Hilfe Cwdaemon.

Nach dem Aufruf von Contestlog muss beim einem CW-Contest immer erst der cwdaemon initialisiert werden. Das erfolgt durch den Aufruf von CWkeyer. Dieser Dialog stellt ein Sortiment von MACROS für die CW-Ausgabe zur Verfügung lädt die Macros und initialisiert den Keyer. Wird die Macro-Ansicht für den laufenden CW-Contest nicht benötigt, dann den Dialog wieder ausschalten.

CWSpeed ändern
WPM / Speed: Der eingestellt Speed steht im Fenste WPM. Mit den Tasten Bild-rauf/runter wird die Einstellung verändert.

F-Tastenbelegung / Macros
Die vordefinierten Macros können jederzeit überschrieben werden. Nach jeder Änderung muss Save durchgeführt werden. Hiermit wird die Änderung gespeichert und zum Contestdialog durchgestellt.

ESC – Wird mit F1 eine CQ Contest Schleife gestartet, kann die Aussendung mit ESC abgebrochen werden.

Rig Control – einbinden
Die Hamlib muss installiert und konfiguriert sein. Wird im Dialog QtLog->Configuriere->Rig Control die angezeigte Frequenz mit OK bestätigt, startet RigCtl automatisch. Der Transceiver muss eingeschaltet sein, sonst gibt es eine Fehlermeldung. Im Dialog Rig Control kann die Hamlib ein -oder ausgeschaltet werden. Außerdem ist der Pollzyklus ( Retry ) änderbar. Siehe Hilfe Rig Cotrol-Hamlib
