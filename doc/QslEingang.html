QSL - Bearbeitung

Die QSL Bearbeitung erfolgt wahlweise in den Bereichen Büro, eQSL oder  LotW.
Je nach Konfiguration im Dialog "QSO Eingabe" wird beim Aufruf der QSL-Bearbeitung  gleich der eingestellte Arbeitsbereiche gewählt. 

 Büro
Mögliche Anzeigen sind: QSL Nicht erhalten, Nicht gedruckt, Storniert, Erhalten.
Ist die angezeigte Liste zu groß oder enthält sie nicht das QSO für die vorliegende QSL-Karte, muss in der Datenbank gesucht werden. 

Es kann nach Call, Mode oder Dxcc gesucht werden.
Nach Eingabe der ersten 2 Buchstaben  wird ab dem 3. automatisch die Datenbankabfrage  gestartet.

Qsl Eingang buchen:
Wird in den Spalten QSLr, QSLs oder prFlg geklickt, meldet sich ein kleiner Eingabe Dialog.

Mögliche Eingaben:
QSLr ( qsl recieved ) : Y = erhalten,   R (requested, gewünscht)   - storno keine QSL.
QSLs ( qsl send  )      : Y = gesendet ( gedruckt ),  R = requested ( soll gedruckt werden ) - keine QSL gewünscht.
prFlg ( printFlag )       : B =  Druckauftrag über Lineprinter, leer = kein Druckauftrag.

QSO ohne gesetzten prFlg ( Printflag ) werden im Druckmodul nicht angezeigt,
Wird in der QSO-Eingabe, QSL gewünscht markiert, wird auch das prFlg gesetzt.
Das prFlg kann jederzeit händig gesetzt oder gelöscht werden.

Award-Kenner eingeben:
Mit dem AWD-Button den Type wählen. Für den DOK -> DLD.
Wird in die Tabellenspalte AWD geklickt öffnet sich die Tabelle für die Awardkenner und ist zur Bearbeitung  freigegeben. 

Nun den Kenner passend zum AWD-Type eingetragen.
Für jedes QSO können beliebige Kenner gebucht werden. Steht der Type noch nicht in der AWD-Tabelle muss dieser erst angelegt werden.
Wird außerhalb der AWD-Spalte geklickt, wird die Tabelle gesperrt, da kein Zusammenhang mehr zur AWD-Tabelle besteht.

 eQSL
Voraussetzung ist ein  eQSL User Name und ein  Passwort. Was im Config eingetragen sein muss.

Wurde im Dialog Qsl Erfassung der Auswahlhaken auf eQSL gestellt, wird beim Aufruf der QSL Bearbeitung gleich der eQSL Arbeitsbereich eingestellt. Das umschalten der Arbeitsbereiche ist aber jederzeit möglich.

Ist der Button eQSL eingestellt stehen 4 Abfragemodi zur Verfügung: Nicht Bestätigt, Nicht gesendet, Gesendet, QSL Erhalten.

Upload zum eQSL.cc Server
Der erste Qso Upload: Den Button 'Nicht gesendet' drücken. Voraussichtlich wird eine große Liste angezeigt. eQSL lässt nur 1000 QSOs je ADIFfile zu. Der QSO Bestand wird daher zeitlich gegliedert. Datum von / bis einstellen und erneut den Button betätigen. Ist die Liste kleiner 1000, wird mit Ok ein ADIF-File erzeugt. Anschließend diese Datei über einen HTML Browser zum Server hochladen, dann die nächste ADIF Datei erzeugen, usw.

Die folgenden QSOs die in der QSO-Erfassung geloggt werden, werden über einen RealTimeUpLoadManager sofort zum Server hoch geladen.
Jedoch nur wenn der Auswahlhaken eQSL gesetzt ist. Wurde dieses vergessen können diese QSOs hier in der QSL Bearbeitung hoch geladen werden. Falls es viele QSOs sind wird wieder ein ADIF-fiele erzeugt und über einen Browser hoch geladen. Wenige QSOs werden mit dem RealtimeUploadMgr einzeln hoch geladen. Wird in ein Rufzeichen geklickt erscheint ein Bestätigungsmenü. Mit Ok, wird das QSO zum Server hoch geladen.

Download vom eQSL.cc Server
Im HeaderMenue Download werden 2 Methoden angeboten. 'eQSL getQSL Recods' und 'eQSL Read Report.adi File'

- eQSL getQSL Records: 
Hier werden 6 Queries zum runter laden bestätigter und nicht bestätigter QSOs angeboten.
Das 1. Query 'All InBox records' ist das wichtigste. Damit kann man täglich oder wöchentlich usw. die eingegangen Bestätigungen abrufen. Die runter geladenen bestätigten QSOs werden angezeigt. Mit 'All Archived records only' kann der eQSL-Datenbestand mit dem Logbook abgestimmt werden.

'Check QSL Card Status'
prüft den Qsl Kartenbestand und schlägt die fehlenden Karten zum Download vor.
Die Qslkarten für bestätigte QSOs werden nicht automatisch runter geladen. Es ist daher zweckmäßig nach jedem Eingang bestätigter QSOs den QSL Kartenbestand abzufragen und die fehlen Karten abzurufen.

- eQSL Read Report.adi File:
Wird eine Adifdatei über den HTML Browser runter geladen. Wird mit diesem Tool diese gelesen und ausgewertet. Das Ergebnis wird angezeigt.

Nach dem Download sollte man sich bei eQSL.cc einloggen und die aus InBox runter geladenen LOG's ins Archiv verschieben.

  LotW  - Logbook of The World
Voraussetzung: Das TrustedQSL-System muss installiert und ein gültiges Zertifikat vorhanden sein.
Informationen hierzu gibt es unter ARRL "The Logbook of the WORLD".

Mit dem Button QSL via den Arbeitsbereiche für LotW einstellt.
Der Dialog wird modifiziert und die Kalender für den Upload Zeitraum freigegeben.

Der linke Kalender zeigt das Tagesdatum von heute. Der rechte, das Datum vom letzten Upload nach LotW.
Das  last Upload Datum wird fortgeschrieben, so das der letzte Upload immer sofort erkennbar ist. Das Startdatum für last Upload  ist auf 1974 gestellt. Das ist ein Beispiel und muss entsprechend korrigiert werden.

QSO  Upload :
Kalender Einstellung prüfen.
Mit dem Button "Nicht gesendet" die nicht gesendeten QSOs abfragen. Beim ersten Aufruf ist es das gesamte Logbook.
Je nach Einstellung der Kalenderdaten, ist es möglich das komplette Logbook oder nur Segmente, ausgehend vom ältesten QSO an LotW zu senden.

Zweckmäßig ist ( zum  kennen lernen ) den Upload nach Jahrgängen, klein, größer und den Rest aufzuteilen. Mit dieser Methode hat man die Möglichkeit den Download in einer überschaubaren Zeiteinheit zu prüfen.

Im Header Menü "Upload" den Button "Lotw QSO Upload" aufrufen.
Es wird jetzt eine Adifdatei mit dem Namen "call_datum.adi" erzeugt. 
Es meldet sich "TrustedQSL" und fragt nach dem Passwort ( wenn ein passwort vergeben wurde ).

TrustedQSL erzeugt die Datei "call_datum.tq8". Es erscheint die Meldung  call_datum.tq8 erzeugt und steht unter (HOME)/lotw zum Upload bereit. Mit einem Web Browser bei ARRL einloggen und die zertifiziert Datei *.tq8 hochladen.

QSL  Download :
Nach ca. 20 Minuten Verarbeitungszeit  kann der ARRL Server  abgefragt werden.

Unterhalb vom Header Popupmenü Download sind die Button :
LOTW read report.adi
LOTW get QSL Reports.

LOTW readreport.adi :
Setzt voraus das die Datei lotwreport.adi über einen Browser bereits  heruntergeladen wurde. Es meldet sich daher ein Datei Selektor zur Selektion der download Datei. Nach der Auswahl wird  der Qslreport verarbeitet und das Ergebnis angezeigt.

LOTW get QSL Records :
Hier meldet sich ein Query-Dialog. Zur Zeit werden 7 Abfrage Methoden angeboten. Die Nr. 1 ist die default Abfrage. Mit den übrigen können QSL für CALL, MODE, BAND DXCC abgefragt werden. Mit Nr. 7 werden alle bestätigten QSO vom Server runter geladen.

Nach der Bestätigen am Button Download wird Kontakt zum ARRL Server hergestellt und die QSL für  die eingestellte Selektion heruntergeladen. Die QSOs mit Bestätigungen werden in der Tabelle dargestellt . Die Anzahl der  QSL-Neuzugänge wird angezeigt.

Mögliche Fehlermeldungen:
- Zeitüberschreitung. Kein Kontakt zum Server.
- User Name oder Passwort falsch.
- Server ist überlastet ( to many users )
 
Hinweise zum TrustedQSL :
Zur Zeit muss TrustedQSL von jedem selbst installiert werden. 

Das qtlog_tararchiv enthält zwar  die Sourcen vom TrastedQSL. Dieser Programmcode ist aber vom Autor nicht gut gepflegt und muss überarbeitet werden. Beim kompilieren entstehen je nach verwendeter Distribution mehr oder weniger große Fehler.
Für SuSE Linux 11 gibt es ein rpm Paket. Bei Bedarf stelle ich es gern zur Verfügung.
Es ist in der Planung TrastedQSL im QtLog fest einzubinden und QSL Download / Upload komplett zu automatisieren.
