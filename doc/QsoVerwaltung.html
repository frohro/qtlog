QSO – Verwaltung

QtLog – Logbook und Informationssystem für den Amateurfunk

Das Logbook unterstützt die Verwaltung von QSO, DXCC-Länder und AWARDS.

Das Programm ist überwiegend selbsterklärend. Leider konnte dieses aber nicht in allen Bereichen erreicht werden. Wichtig ist, schnell eine Programmübersicht zu erhalten.
Hierfür gibt es für die  meisten Dialoge HILFE-Seiten mit direktem Zugriff. Es stehen Erläuterungen zur Abfragemethode und Datendarstellung zur Verfügung.

Als Datenbasis wird die MySQL-Datenbank verwendet.
Das Log Programm besteht aus einem Hauptprogramm und mehreren Nebenprogrammen.
Wird QtLog aufgerufen, meldet sich der Dialog "QSO-Verwaltung".
Von hier können alle übrige Funktionen aufgerufen werden.

Dialog QSO - Verwaltung:
-----------------------------------

In den Header -Auswahl-Menüs werden alle benötigten Methoden zu Abfrage und Verwaltung der QSO-Daten angeboten. Hilfe zum Verständnis und zur Handhabung der Daten sind in fasst allen Dialogen vorhanden.

Als erstes sollte das System configuriert werden. Entweder 'Konfiguriere' direkt aufrufen oder hier im Manual zur Information die Seite 'Konfiguriere' klicken.

Das Programm QtLog wird in 3 unterschiedlichen Config Bereichen eingestellt.
Logbook : Es können mehrere Logbooks verwaltet werden. Angeboten wird das Logbook fun zum testen. Ist das Programm ausreichend gestetet, wird das eigene Logbook angelegt.
Stations Standort : Hier werden die persönliche Daten eingetragen. Diese Parameter werden vom Programm in vielen Bereichen benötigt.
Club :  wie bekannt.

Die QSO Verwaltung ist in erster Linie eine Informationsquelle zu den geführten QSOs.
Die Abfrage der QSO-Daten von der Datenbank erfolgt über Datenbankfilter.
Das Ergebnis wird in einer Report-Tabelle angezeigt.

Der Datenbankfilter besteht aus 6 Gruppen.
    Ausgabebegrenzung:  max. 100 bis 2000 Qsos je Tabelle. - Empfehlung 400 Qsos.
    Abfragekriterien: Call, Band, Mode usw. ...
    Abfrage der Award Kenner: Alle konfigurierten AWD-Typen.
    Zeitraum Vorgaben: Default ist das laufende Jahr. Der Zeitraum ist beliebig wählbar.
    Daten-Sortierung: Datum, Band, Call, DXCC
    Report-Layout: Freie Auswahl der darzustellenden Tabellen-Spalten. 
    Alle Datenbankfelder  sind zugänglich und darstellbar.

Hinweise zu einigen Merkmalen der Datenbankfilters:
---------------------------------------------------------------------
Anzeige max.: Die Anzeige 2000 Datensätze sollte nicht die Regel sein. Viele Datensätze bedeuten auch lange Wartezeiten bis zur Ausgabe. 200/400 Datensätze sind im allgemeinen ausreichend. Größere Datenmengen lassen sich auch schwer übersehen. Besser ist es die Auswahlkriterien  entsprechend  zu wählen.

Anzeige ohne/mit Zeitraum : Ist der Haken 'Zeitraum' nicht gesetzt ist die Zeitraumbegrenzung ausgeschaltet.
Bei gesetztem Haken wirkt der voreingestellt Zeitraum. Das Datum 'Von' ist das aktuelle Tagesdatum. Das Datum 'Bis' steht defaultmäßig immer auf den 1.Januar des laufenden Jahres. Stellt man den Textcursor in das Feld 'suche' ohne Texteingabe und wird der Zeilenvorschub gedrückt,  werden alle Qsos des laufenden Jahres angezeigt.
Die Qsoausgabe mit Zeitraum kann Tag genau abgerufen werden.

ReportLayout: Die Form der Datendarstellung kann jeder Anwender selbst konfigurieren. Im Headermenü "Logbook - Verwaltung"  die Methode "ReportLayout" aufrufen. In diesem Menü werden die gewünschten Spalten der Tabellen zusammengestellt. Die Tabellenspalte wird mit einem  Klick in den Log-Datensatz gewählt oder entfernt. Die Reihenfolge im Fenster 'Anzeige' entspricht der Spaltenfolge in der Tabelle.

Es sind 4 vorkonfigurierte Layouts möglich. EU, DX, QSL und Var ( wie variable ).
Diese Konfiguration kann der Anwender jeder zeit verändern,
Beispiel: 
- Layout 'EU' wichtige Merkmale  für Qsos aus Europa z.B. WAE, DLD usw. 
- Layout DX entsprechend. 
- QSL Datenspalte für die QSL Übersicht ( QSL via Büro, LotW .. usw )  
- Var  zur Forschung in der Datenbank, 

Ziel sollte sein die Spaltenanzahl in der Tabelle so gering wie möglich zu halten. Viele Spalten benötigen auch viel Zeit für den Tabellenaufbau.


Daten abfragen:
----------------------
Zum suchen nach den gewünschten QSO-Daten gibt es die Filter :  'Suche' und 'Zeige AWARD-Typen'.

Filter Suche :
Wird in das Eingabefeld unterhalb des Auswahlbuttons kein Kriterium  angegeben und  'cr' ( Zeilenvorschub ) gedrückt, werden alle Qsos die in der Datenbank vorhandenen sind angezeigt, jedoch max. 2000. Diese Datenmenge lässt sich nicht überblicken und ist daher wenig sinnvoll.

Die Eingrenzung der Anzeige erfolgt durch Argumente im Auswahl-Button z.B 'Call' . Das Eingabefeld hat eine Match-Funktion. Wird  nur 'dk' oder 'DK' eingegeben werden alle Qsos die mit dem Rufzeichen DK beginnen angezeigt usw.

Filter AWARD Type:
Beim Filter Zeige Award Typen ist es ähnlich. Ist kein Auswahlwort angegeben werden alle Qsos mit dem Kenner gemäß eingestelltem Type angezeigt.

Beispiel : Es soll der Kenner E33 abgefragt werden. Hierfür wird der AWD Type auf DLD gestellt, im Eingabe-Feld der Kenner  'E33' eingetragen. Kenner werden immer komplett angegeben. Award-Typen und Kenner können nur angezeigt werden, wenn im Reportlayout die Spalte 'AWD' eingebunden ist.

Die Daten im Report bearbeiten:
--------------------------------------------
Alle Felder in der QSO-Tabelle sind editierbar. Mit einem Doppelklick in die Datenzelle öffnet sich der Texteditor. 
Die Texteingabe wird mit 'cr' beendet. 
Mit 'ESC' kann der Editor ohne Textspeicherung verlassen werden. 

Der neue Text wird gespeichert. Mit einem weiterem 'ESC' wird die Zellenselektion aufgehoben ( der blaue Hintergrund ist dann weg ).

AWARD-Kenner bearbeiten: ( beta Version )
Ist in der Report-Tabelle eine AWARD-Spalte eingebunden z.B DLD. Werden in dieser Spalten die eingeloggten 'DOK' angezeigt. Wird auf einen DOK geklickt öffnet sich ein DockWidget und es werden alle für dieses QSO eingeloggten Kenner dargestellt. Ein Klick in eine nicht AWD Tabellenzelle schließt das DockWidget.

Im geöffneten DockWidget werden in der linken Spalte alle konfigurierten AwardTypen abgebildet. Die rechte Spalte enthält die Kenner zu diesem Qso. Es können vorhandene Kenner verändert oder neue eingetragen werden. Neue AWD-Typen können hier nicht eingerichtet werden.

Wird in der AWD-Tabelle ( DockWidget ) in den Doppelrahmen rechts oben  geklickt schwebt die AWARD-Tabelle über dem Report. Die Größe ist jetzt änderbar. Lässt sich aber leider noch nicht auf dem Bildschirm verschieben. Die Report-Tabelle kann unterhalb des DockWidget verschoben werden. Klickt man mit der Maus in eine nicht AWD-Zelle wird das AwardFenster geschlossen weil der Bezug zum AWD nicht mehr vorhanden ist.
